# YouTube to Presentation Generator

本ツールは、YouTube動画のURLを入力するだけで、自動的に文字起こし（字幕）と高画質なスライド画像（動画のフレーム）を抽出し、PowerPointプレゼンテーション（.pptx）を自動生成するローカルWebアプリケーションです。

## 主な機能
- **高画質スライド抽出 (1080p対応)**: スライド画像を1080pの高精細な画質でダウンロードし、スライドを自動生成します。AV1デコードエラーを避けるため、互換性の高い H.264 コーデックを自動で選択します。
- **Closed GOP Iフレーム参照**: ffmpeg/ffprobe を用い、Closed GOP の Iフレーム（キーフレーム）のみを参照して、基準フレームとの差分からスライド切替を検出します。
- **変化レベルの調整**: 10段階の変化感度スライダーを操作し、わずかな変化から大きなスライドの切り替わりまで、意図した感度（MAD閾値: 1.0 〜 100.0）でスライドを抽出できます。
- **自動字幕取得**: 日本語（手動 → 自動）および英語（手動 → 自動）などの字幕を優先順位に従って自動取得し、画像とマッピングします。字幕が取得できない動画でも、画像のみのスライド生成を続行します。
- **動画のダウンロードキャッシュ**: 同一URLでの生成パラメータ調整時には動画の再ダウンロードをスキップするキャッシュ機能を搭載しています。
- **不要な一時ファイルの自動クリーンアップ**: 新しい動画URLの処理開始（URL更新時）およびアプリケーション終了時に、不要な一時ファイルを自動でクリーンアップします。
- **トースト通知の統合**: 画面上の通知システムをスタック表示可能なトースト通知に統一し、進捗状況やエラーを視覚的に把握しやすくなりました。

## 動作要件
- OS: Windows 10 / 11, Linux (Ubuntu等)
- Python: 3.8 以上 (環境変数 PATH に `python` または `python3` が登録されている必要があります)

## セットアップと起動手順

### Linux 環境の場合

#### 1. セットアップ（初回のみ）
1. 本プロジェクトのルートディレクトリを開きます。
2. ターミナルでセットアップスクリプトを実行します：
   ```bash
   ./setup.sh
   ```
3. 仮想環境（`.venv`）が自動で作成され、必要な依存パッケージや `Playwright` ブラウザコンポーネント、および `yt-dlp` が自動ダウンロードされます。

#### 2. アプリケーションの起動
1. ターミナルで起動スクリプトを実行します：
   ```bash
   ./run.sh
   ```
2. Webサーバーが起動し、自動的にブラウザが開いて `http://localhost:8000` にアクセスします。
3. 終了する場合は、ターミナルで `Ctrl + C` を押してください。

---

### Windows 環境の場合

#### 1. セットアップ（初回のみ）
1. 本プロジェクトのルートディレクトリを開きます。
2. `setup.bat` をダブルクリックして実行します。
3. 自動的に仮想環境（`.venv`）が作成され、必要なパッケージと `Playwright` 関連のブラウザコンポーネントがインストールされます。

#### 2. アプリケーションの起動
1. 本プロジェクトのルートディレクトリにある `run.bat` をダブルクリックして実行します。
2. Webサーバーが起動し、自動的にブラウザが開いて `http://localhost:8000` にアクセスします。
3. 終了する場合は、起動したコマンドプロンプトのウィンドウを閉じるか、`Ctrl + C` を押してください。

---

## 使い方
1. ブラウザに表示された画面で、解析したい **YouTube 動画 URL** を入力します。
2. 必要に応じて **変化検知レベル**（1：非常に敏感 〜 10：最も鈍感）をスライダーで調整します。
3. **「プレゼンテーションを生成する」** ボタンをクリックします。
4. 処理が完了すると、画面下部に生成されたスライドのプレビューが表示され、**「PowerPoint をダウンロード」** ボタンから PPTX ファイルを取得できます。

### 変化レベル（感度）とMADしきい値の対応
スライド切り替えの判定基準（MAD値）の10段階の対応は以下の通りです。デフォルトは **レベル 5 (MAD閾値 15.0)** です。

| 変化レベル (感度) | MAD 閾値 (Threshold) | 特徴 |
| :---: | :---: | :--- |
| **1** | **1.0** | 非常に敏感（わずかな変化も検知） |
| **2** | **3.0** | |
| **3** | **6.0** | |
| **4** | **10.0** | |
| **5 (デフォルト)** | **15.0** | **標準設定** |
| **6** | **25.0** | |
| **7** | **40.0** | |
| **8** | **60.0** | |
| **9** | **80.0** | |
| **10** | **100.0** | 最も鈍感（大きな変化のみ検知） |

## ディレクトリ構造
- `app/` : アプリケーションのソースコード
  - `main.py` : FastAPIを用いたサーバーエントリーポイント
  - `services/` : ビジネスロジック（動画処理、画像抽出、PPTX生成など）
  - `static/` : フロントエンドのUIリソース（HTML/CSS/JS）
- `setup.bat` / `setup.sh` : 初期セットアップ用スクリプト
- `run.bat` / `run.sh` : サーバー起動用スクリプト
- `requirements.txt` : 依存パッケージの定義ファイル

## ✨ Iフレーム抽出の高速化（v2.0）
Iフレーム抽出処理を大幅に最適化し、処理時間を **40～50% 削減** しました。

### 主な改善点
- **ビデオスキャン統一**: 動画の読み込みを2回 → 1回に削減
- **フレームキャッシング**: Iフレーム処理時にメモリに保持して再ロード不要化
- **ffprobe最適化**: タイムアウト短縮とフォールバック処理改善
- **進捗表示強化**: リアルタイムで処理状況を表示

### 期待される効果
- 30分動画：処理時間 **3～5分 → 1.5～2.5分**
- 大規模動画ほど効果的（ビデオI/Oの削減が大きい）

詳細な最適化レポートは [OPTIMIZATION_REPORT.md](./OPTIMIZATION_REPORT.md) を参照してください。
