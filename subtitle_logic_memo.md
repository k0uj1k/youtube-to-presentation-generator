# 字幕（文字起こし）取得ロジックの改善メモ

`subtitle_service.py` における字幕取得処理を安定化・高機能化するための設計メモです。

## 検討された字幕取得の基本方針

### 1. 言語の確認
- `yt-dlp` を用いて、対象動画に登録されている字幕（手動字幕および自動生成字幕）の言語一覧を確認します。
- この時点で字幕データが一切存在しない（音声認識すらされていない）場合は、文字起こしの取得をあきらめ、画像のみでスライドを生成します。

### 2. 日本語の字幕がある場合
- 日本語の手動字幕、または日本語の自動生成字幕を優先的に取得します。

### 3. 日本語以外の字幕がある場合
- **日本語に自動翻訳できるか？** を確認します。
  - **できない場合**: 原文の言語のまま文字起こしを取得します。
  - **できる場合**: 日本語に自動翻訳したテキストを取得します。

---

## 技術的な実装アプローチ

YouTube の仕様変更（ timedtext API の直接リクエストに対するシグネチャ制限など）に対応するため、以下の方法で実装します。

### 1. `yt-dlp` でのメタデータ解決
- `yt-dlp.YoutubeDL` の `extract_info(url, download=False)` を呼び出し、動画に紐づく字幕のメタデータ（`subtitles` と `automatic_captions`）を取得します。
- これにより、安全に署名された字幕のダウンロード用 URL（VTT 形式）が手に入ります。

### 2. YouTube 自動翻訳機能の利用
- 日本語以外の字幕を日本語に翻訳して取得する場合、取得した他言語字幕の URL の末尾に `&tlang=ja` パラメータを追加して HTTP リクエストを送ることで、YouTube サーバー側で翻訳された VTT データを取得できます。

### 3. WebVTT パーサーの実装
- メモリ上で VTT テキストを解析し、スライドマッピングに必要な形式 `[{"start": 開始秒数, "text": 字幕テキスト}]` に変換します。
- 自動生成字幕に特有の HTML タグ（`<c>` など）を正規表現でクリーンアップします。
